1. Mental Model

These operators combine truthiness evaluation with control flow in a single expression:

  • Logical operators (&&, ||) return the value of an operand, not a boolean
  • Short-circuit evaluation stops evaluating when the result is determined
  • Null coalescing (??) specifically targets null and undefined
  • Optional chaining (?.) safely accesses nested properties
Key insight: These operators are expressions, not just control flow. They return actual values that can be assigned or used further.

2. Logical Operators

Logical AND (&&)

Returns the first falsy operand, or the last operand if all are truthy.

true && "hello";     // "hello"
false && "hello";    // false
5 && 10;             // 10
0 && 10;             // 0
"" && "hello";       // ""
null && 5;           // null

Logical OR (||)

Returns the first truthy operand, or the last operand if all are falsy.

false || "hello";    // "hello"
"" || "default";     // "default"
0 || 42;             // 42
5 || 10;             // 5
null || undefined;   // undefined
false || 0 || null;  // null

3. Short-Circuit Evaluation

The && Short-Circuit

The right operand is only evaluated if the left operand is truthy. 

const obj = null;
obj && obj.name;  // null (obj.name is never evaluated)

const user = { name: "Alice" };
user && user.name;  // "Alice"

The || Short-Circuit

The right operand is only evaluated if the left operand is falsy. 

const config = null;
config || { color: "blue" };  // { color: "blue" }

const theme = { color: "red" };
theme || { color: "blue" };   // { color: "red" }

Side Effects and Evaluation

Short-circuit evaluation prevents unnecessary function calls and side effects: 

function expensiveCall() {
  console.log("Running!");
  return 42;
}

false && expensiveCall(); // "Running!" never logs

true || expensiveCall();  // "Running!" never logs
Performance implication: Right operands are not evaluated if the short-circuit condition is met. This can be used intentionally to avoid expensive operations.

4. Null Coalescing Operator (??)

What It Does

Returns the right operand only if the left is null or undefined. All other falsy values (0, "", false, NaN) are treated as valid. 

null ?? "default";       // "default"
undefined ?? "default";  // "default"
0 ?? "default";          // 0
"" ?? "default";         // ""
false ?? "default";      // false
NaN ?? "default";        // NaN

Real-World Example

// Configuration with valid falsy values
const config = {
  timeout: 0,       // 0 is valid!
  retries: false,   // false is valid!
  name: null        // null means use default
};

const timeout = config.timeout ?? 30;    // 0 (valid)
const retries = config.retries ?? 3;     // false (valid)
const name = config.name ?? "Guest";     // "Guest" (null, use default)
Use case: When 0, "", or false are valid values, prefer ?? over ||.

5. Optional Chaining (?.)

Property Access

Returns undefined if the left side is null or undefined, otherwise accesses the property. 

const user = null;
user?.name;  // undefined

const person = { name: "Alice" };
person?.name;  // "Alice"

Method Calls

const obj = { greet: () => "Hello" };
obj?.greet();  // "Hello"

const empty = null;
empty?.greet();  // undefined (method not called)

Array/Computed Access

const arr = [1, 2, 3];
arr?.[0];  // 1

const empty = null;
empty?.[0];  // undefined

Chaining Multiple Levels

const response = {
  user: {
    profile: {
      name: "Alice"
    }
  }
};

response?.user?.profile?.name;  // "Alice"

const partial = { user: null };
partial?.user?.profile?.name;   // undefined (stops at null)
Short-circuit behavior: Once ?. encounters null or undefined, the entire chain returns undefined and further access is skipped.

Combining with Parentheses

// Mixing && and ||
const value = false && true || "default";
// Evaluates: (false && true) || "default" = "default"

// Null coalescing with ||
const x = null || 5 ?? 10;
// Evaluates: (null || 5) ?? 10 = 5 ?? 10 = 5

// Better: use parentheses for clarity
const config = (null ?? "default") || "fallback";
// Evaluates: "default" || "fallback" = "default"
Caution: Mixing &&/|| with ?? without parentheses raises a SyntaxError. Use explicit grouping.

7. Common Pitfalls

Mistake 1: Using || When You Mean ??

// Wrong: will use default for valid 0
const count = userInput.count || 10;

// Right: 0 is a valid count
const count = userInput.count ?? 10;

Mistake 2: Not Grouping Mixed Operators

// SyntaxError: cannot mix && and ?? without parentheses
const x = a && b ?? c;

// Correct
const x = (a && b) ?? c;

Mistake 3: Forgetting Short-Circuit Side Effects

let result;
false && (result = "assigned");  // result stays undefined!

const value = result || "default";  // "default"

Mistake 4: Assuming ?. Returns Boolean

const obj = null;
if (obj?.name) { }  // Falsy check works, but returns undefined (not false)

// More explicit:
if (obj?.name !== undefined) { }

Mistake 5: Optional Chaining on Non-Nullish Values

const str = "hello";
str?.length;  // 5 (works fine)

const num = 0;
num?.toFixed(2);  // Throws: 0 is valid, but doesn't have toFixed
// Optional chaining doesn't prevent errors for invalid method calls
Insight: ?. checks for null/undefined, not whether the property exists or is callable.

8. Real-World Patterns

API Response Handling

const response = await fetch("/api/user");
const user = await response.json();

// Safe navigation through nested structure
const username = user?.profile?.name ?? "Anonymous";
const avatar = user?.profile?.avatar ?? "/default.png";

Default Values with Valid Falsy

// Form submission with 0 and empty string as valid
const formData = {
  count: input.count ?? 0,       // User can submit 0
  tags: input.tags ?? [],        // User can submit []
  approved: input.approved ?? false  // User can submit false
};

Conditional Execution

// Run callback only if it exists
user?.onLogout?.();

// Chain operations safely
const config = loadConfig()
  ?.validate()
  ?.toJSON();

// Defaults at each step
const fontSize = config?.fontSize ?? 14;

Combining for Robustness

const getTimeout = (config) => {
  return config?.timeout ?? globalConfig.timeout ?? 30000;
};

const getColorMode = (prefs) => {
  return prefs?.color && (prefs.color === "dark" ? "dark" : "light")
    ?? systemDefault.color;
};

9. Best Practices

Do

  • Use ?? for null/undefined defaults
  • Use ?. to safely navigate nullable structures
  • Use && for guarded execution (only if truthy)
  • Use parentheses when mixing &&/|| with ??
  • Prefer ?? for configuration with falsy valid values (0, "")

Don't

  • Use || for null defaults when 0 or "" are valid
  • Chain ?. excessively (consider destructuring instead)
  • Assume ?. validates method callability
  • Use && for side effects in complex expressions
  • Mix &&/|| with ?? without parentheses

Code Examples

// Good: explicit intent
const timeout = config?.timeout ?? 5000;
const name = user?.name ?? "Guest";

// Bad: implicit fallback
const timeout = config?.timeout || 5000;
const name = user?.name || "Guest";
Rule of thumb: Read the operator as a sentence. ?? = "or use this if null/undefined" || = "or use this if falsy" && = "and then if truthy"

10. Interview Questions

Q: What does null && undefined return?

null && undefined
// Returns: null (first falsy value)

Q: What's the difference between these?

false || 0;     // 0
false ?? 0;     // false
false && 0;     // false

Q: When does obj?.method() NOT call the method?

const obj = null;
obj?.method();  // undefined, method not called

const obj = { method: null };
obj?.method();  // null (method is null, not called)

Q: What's the result?

const a = 0;
const b = null;
const result = (a ?? 10) || (b ?? 20);
// a ?? 10 evaluates to 0
// 0 || (b ?? 20) evaluates to 0 || 20 = 20
// Result: 20

Q: Why does this throw an error?

const x = a && b ?? c;
// SyntaxError: missing parentheses
// Fix: const x = (a && b) ?? c;

Q: What's printed?

let called = false;
const fn = () => (called = true, "value");

const result = false && fn();
console.log(called);  // false (fn never called)
