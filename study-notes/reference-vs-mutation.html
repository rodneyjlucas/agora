1. Concept and Example

JavaScript variables store values. For primitive values, the value itself is copied. For objects, arrays, and functions, the value copied is a reference to an object.

Core idea: JavaScript is pass-by-value, but object values are references. That means the reference is copied, not the object itself.
Kind of valueExamplesAssignment behavior
Primitivenumber, string, boolean, null, undefined, symbol, bigintCopies the primitive value.
Object{}, [], functions, dates, maps, setsCopies a reference to the same object.

Primitive copy

let a = 10;
let b = a;

b = 20;

console.log(a); // 10
console.log(b); // 20

Changing b does not affect a because the number value was copied.

Reference copy

const obj1 = { count: 1 };
const obj2 = obj1;

obj2.count = 5;

console.log(obj1.count); // 5
console.log(obj2.count); // 5

Both variables point at the same object, so mutating through one variable is visible through the other.

Mutation vs reassignment

Mutation

const user = { name: "Ada" };

user.name = "Grace";

console.log(user);
// { name: "Grace" }

Mutation changes the existing object.

Reassignment

let user = { name: "Ada" };

user = { name: "Grace" };

console.log(user);
// { name: "Grace" }

Reassignment makes the variable point to a different object.

Important distinction: const prevents reassignment of the variable binding. It does not freeze or protect the object from mutation.

3. Code Examples

3.1 Array reference mutation

const arr1 = [1, 2, 3];
const arr2 = arr1;

arr2.push(4);

console.log(arr1); // [1, 2, 3, 4]
console.log(arr2); // [1, 2, 3, 4]

arr1 and arr2 reference the same array. push mutates that shared array.

3.2 Avoid mutation with a shallow copy

const arr1 = [1, 2, 3];
const arr2 = [...arr1];

arr2.push(4);

console.log(arr1); // [1, 2, 3]
console.log(arr2); // [1, 2, 3, 4]

The spread syntax creates a new top-level array, so arr2.push() does not change arr1.

3.3 Function parameters can mutate objects

function increment(obj) {
  obj.value++;
}

const data = { value: 1 };

increment(data);

console.log(data.value); // 2

The function receives a copy of the reference. It cannot reassign the caller's variable, but it can mutate the object the reference points to.

3.4 Return a new object instead

function increment(obj) {
  return { ...obj, value: obj.value + 1 };
}

const data = { value: 1 };
const updated = increment(data);

console.log(data.value);    // 1
console.log(updated.value); // 2

This is an immutable update pattern. The original object remains unchanged.

3.5 Shallow copy with nested objects

const original = {
  user: { name: "Ada" }
};

const copy = { ...original };

copy.user.name = "Grace";

console.log(original.user.name); // "Grace"

The outer object was copied. The nested user object was still shared.

4. Gotchas and Non-intuitive Examples Explained

4.1 const does not mean immutable

const obj = { value: 1 };

obj.value = 2; // allowed
obj = {};      // TypeError
Why: const locks the variable binding. It does not lock the object behind the reference.

4.2 Some array methods mutate and some do not

Mutates original arrayDoes not mutate original array
push, pop, shift, unshiftmap, filter, reduce
sort, reverse, splice, fillslice, concat, toSorted, toReversed, toSpliced
 
const nums = [3, 1, 2];

nums.sort();

console.log(nums); // [1, 2, 3] - original changed

sort() is a common surprise because many developers expect it to return a sorted copy. Use toSorted() or [...nums].sort() when you need to preserve the original.

4.3 Object equality checks identity, not structure

const a = { x: 1 };
const b = { x: 1 };
const c = a;

console.log(a === b); // false
console.log(a === c); // true

a and b look the same, but they are different objects. a and c point to the same object.

4.4 Reassigning a parameter does not reassign the caller's variable

function replaceUser(user) {
  user = { name: "Grace" };
}

const person = { name: "Ada" };
replaceUser(person);

console.log(person.name); // "Ada"

The parameter user is a local variable. Reassigning it only changes the local binding inside the function.

4.5 Mutating a parameter does affect the original object

function renameUser(user) {
  user.name = "Grace";
}

const person = { name: "Ada" };
renameUser(person);

console.log(person.name); // "Grace"

This time the function did not reassign the local variable. It followed the reference and changed the object itself.

4.6 Spread is shallow

const state = {
  profile: { name: "Ada" },
  loggedIn: true
};

const nextState = { ...state };
nextState.profile.name = "Grace";

console.log(state.profile.name); // "Grace"

A shallow copy copies one level. Nested objects still need their own copies if you want true immutability for that path.

4.7 Correct nested immutable update

const state = {
  profile: { name: "Ada" },
  loggedIn: true
};

const nextState = {
  ...state,
  profile: {
    ...state.profile,
    name: "Grace"
  }
};

console.log(state.profile.name);     // "Ada"
console.log(nextState.profile.name); // "Grace"

5. Best Practices

Prefer immutable updates for app state

const updatedUser = {
  ...user,
  name: "Grace"
};

This makes data flow easier to reason about, especially in React, reducers, and tests.

Avoid mutating function arguments

function rename(user, name) {
  return { ...user, name };
}

Returning a new value makes the function safer and more predictable.

Use modern non-mutating array methods when available

const nums = [3, 1, 2];

const sorted = nums.toSorted();
const reversed = nums.toReversed();
const withoutMiddle = nums.toSpliced(1, 1);

console.log(nums); // [3, 1, 2]

Use structuredClone for deep copies when appropriate

const original = {
  user: { name: "Ada" },
  tags: ["js", "state"]
};

const copy = structuredClone(original);

copy.user.name = "Grace";

console.log(original.user.name); // "Ada"
Note: Deep cloning is not always the best answer. It can be more expensive than copying only the parts you need to update.

Freeze configuration objects when mutation would be dangerous

const config = Object.freeze({
  apiBaseUrl: "/api/v1"
});

config.apiBaseUrl = "/api/v2"; // ignored or throws in strict mode

Object.freeze is shallow. Nested objects need to be frozen separately if you need deep protection.

6. Summary

Primitive values

Numbers, strings, booleans, null, undefined, symbols, and bigints are copied as values.

Objects

Objects, arrays, and functions are accessed through references. Assignment copies the reference.

Mutation

Mutation changes an existing object. Every variable referencing that object can observe the change.

QuestionAnswer
Does assignment copy an object?No. It copies the reference.
Does const make an object immutable?No. It only prevents reassignment of the binding.
Does spread make a deep copy?No. Spread is shallow.
Can functions mutate passed objects?Yes, if they modify the object through the copied reference.
How do you avoid shared mutation?Create a new object or array for the parts you need to change.
Final rule: Primitive → value copy. Object → reference copy. Mutation → shared effect. Reassignment → new binding target.
