1. Summary

this is not the function's owner. It is usually chosen by how the function is called. That tiny detail has consumed entire afternoons from developers who could have been outside pretending nature is still fine.

this

The value available inside a function body. For normal functions, it is usually determined by the call site.

call()

Immediately invokes a function while explicitly supplying the this value and arguments.

bind()

Does not invoke immediately. It returns a new bound function with this and optional leading arguments locked in.

Detached method problem

"use strict";

const user = {
  name: "Ada",
  sayName() {
    console.log(this.name);
  }
};

const detached = user.sayName;
detached(); // TypeError: Cannot read properties of undefined

The call changed from user.sayName() to detached(). There is no object before the dot anymore, so strict mode gives the function this === undefined.

4. call()

call() invokes a function immediately. The first argument becomes this; the remaining arguments are passed to the function normally.

function introduce(greeting, punctuation) {
  return `${greeting}, I am ${this.name}${punctuation}`;
}

const ada = { name: "Ada" };
const grace = { name: "Grace" };

console.log(introduce.call(ada, "Hello", "."));
console.log(introduce.call(grace, "Hi", "!"));
Hello, I am Ada. Hi, I am Grace!

Borrowing methods with call()

A common pattern is to borrow a method from one object and run it with another object as this.

const reporter = {
  report() {
    return `${this.name}: ${this.status}`;
  }
};

const task = {
  name: "Deploy",
  status: "ready"
};

console.log(reporter.report.call(task)); // "Deploy: ready"

5. bind()

bind() returns a new function. It does not run the original function immediately. The new function remembers the chosen this value.

function sayName() {
  console.log(this.name);
}

const user = { name: "Ada" };

const boundSayName = sayName.bind(user);

boundSayName(); // "Ada"

This is useful when a function will be called later by someone else, such as an event system, a timer, or a callback API.

Preset arguments with bind()

function multiply(a, b) {
  return a * b;
}

const double = multiply.bind(null, 2);

console.log(double(5));  // 10
console.log(double(12)); // 24

Here this is irrelevant, so null is supplied. The first argument, 2, is preset. Later calls supply the remaining argument.

Binding methods for callbacks

const button = {
  label: "Save",
  click() {
    console.log(`Clicked ${this.label}`);
  }
};

const handler = button.click.bind(button);

setTimeout(handler, 1000); // later: "Clicked Save"

6. call vs bind

The difference is timing. call() is “run this now with this value.” bind() is “make me a reusable function that will run later with this value.” Ridiculously small wording, radically different behavior. Naturally.

call(): immediate invocation

function show() {
  console.log(this.name);
}

show.call({ name: "Ada" });
// runs now

bind(): delayed invocation

function show() {
  console.log(this.name);
}

const showAda = show.bind({ name: "Ada" });
showAda();
// runs later

7. Gotchas and Non-Intuitive Details

7.1 Arrow functions ignore call() and bind() for this

const user = { name: "Ada" };

const arrow = () => this.name;

console.log(arrow.call(user));
// Does not force this to user.

Arrow functions do not have their own this. Their this is taken from the surrounding lexical scope. call() and bind() can still pass arguments, but they cannot replace arrow this.

7.2 Binding a function twice does not replace the first binding

function showName() {
  console.log(this.name);
}

const ada = { name: "Ada" };
const grace = { name: "Grace" };

const boundOnce = showName.bind(ada);
const boundTwice = boundOnce.bind(grace);

boundTwice(); // "Ada"

Once a function is bound, rebinding the bound function does not change its original bound this. The first binding wins. JavaScript does occasionally commit to something.

7.3 bind() changes function identity

function handleClick() {}

const boundA = handleClick.bind(null);
const boundB = handleClick.bind(null);

console.log(boundA === boundB); // false

Every call to bind() creates a new function. This matters when adding and removing event listeners.

7.4 Do not bind inline when you need to remove the listener later

// Problem: these are two different function objects.
button.addEventListener("click", obj.handle.bind(obj));
button.removeEventListener("click", obj.handle.bind(obj));

// Better: save the bound function.
const boundHandle = obj.handle.bind(obj);
button.addEventListener("click", boundHandle);
button.removeEventListener("click", boundHandle);

7.5 Constructor calls have special rules

function User(name) {
  this.name = name;
}

const BoundUser = User.bind({ name: "Ignored" });
const user = new BoundUser("Ada");

console.log(user.name); // "Ada"

When a bound function is used with new, the newly created object becomes this. The bound thisArg is ignored for construction, though preset arguments still apply.

8. Best Practices

Use method syntax when the function depends on the object before the dot: user.sayName().
Use call() when you want to invoke a function once with a specific this value.
Use bind() when the function will run later and must keep a specific this value.
Do not use arrow functions as object methods when you need this to be the object before the dot.
Avoid repeated inline bind() in render loops or event listener setup. Store the bound function once.

9. Practice: Predict the Output

Question 1

"use strict";

const user = {
  name: "Ada",
  sayName() {
    console.log(this.name);
  }
};

const fn = user.sayName;
fn.call({ name: "Grace" });
Answer: Grace. call() supplies the explicit this.

Question 2

const user = {
  name: "Ada",
  regular() {
    return this.name;
  },
  arrow: () => this.name
};

console.log(user.regular());
console.log(user.arrow());
Answer: user.regular() returns Ada. user.arrow() does not get this from user, so the result depends on the outer scope and is often undefined in module/strict contexts.

Question 3

function greet(word, mark) {
  return `${word} ${this.name}${mark}`;
}

const greetAda = greet.bind({ name: "Ada" }, "Hello");

console.log(greetAda("!"));
Answer: Hello Ada!. bind() fixed this and preset the first argument.

Conclusion

The practical rule is simple: regular functions get this mostly from how they are called. call() invokes immediately with an explicit this. bind() creates a reusable function with this attached. Arrow functions opt out of this whole mess by capturing this lexically, which is either elegant or another way to ruin your afternoon, depending on where you put them.

Final rule: Use call() for immediate borrowing or invocation. Use bind() for delayed callbacks. Use method syntax when this should mean “the object before the dot.”

Core Concept: Understanding JavaScript this

The most important concept behind call, apply, and bind is understanding how JavaScript determines the value of this.

Critical Rule: Regular functions receive this from HOW they are called. Arrow functions capture this lexically from where they were created. 
const user = {
  name: "Ada",

  regular() {
    console.log(this.name);
  },

  arrow: () => {
    console.log(this.name);
  }
};

user.regular(); // "Ada"
user.arrow();   // usually undefined

Humans keep expecting arrow functions to behave like object methods because apparently suffering was not enough and JavaScript needed to help. Arrow functions do not create their own this.

Why call/apply/bind Matter

function greet(message) {
  console.log(message, this.name);
}

const user = { name: "Grace" };

greet.call(user, "Hello"); // Hello Grace

call, apply, and bind exist primarily to control the value of this for regular functions.

Important: Arrow functions ignore rebinding attempts from call, apply, and bind.
