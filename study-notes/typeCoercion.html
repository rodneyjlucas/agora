1. Mental Model

Type coercion is the conversion of values between types using ECMAScript-defined rules.

It occurs in:

  • Operators (+, ==, <)
  • Control flow (if, while)
  • Built-ins (Number(), String())
Key idea: JavaScript coercion is not random. It is rule-based, but often non-intuitive.

2. Core Abstract Operations

ToPrimitive

Converts objects to primitives.

const obj = {
  [Symbol.toPrimitive](hint) {
    return hint === "number" ? 10 : "hello";
  }
};

+obj;        // 10
String(obj); // "hello"

3. Operator Coercion

Addition (+)

1 + 2;       // 3
"1" + 2;     // "12"
1 + "2";     // "12"

Rule:

  • If either operand is a string → concatenate
  • Otherwise → numeric addition

Equality (==)

[] == false;        // true
"" == 0;            // true
null == undefined;  // true

Strict Equality (===)

0 === false; // false
"" === 0;    // false

Relational Operators

"2" < 10;   // true
"10" < "2"; // true (string comparison)

4. Object Coercion

const obj = {
  valueOf() { return 10; },
  toString() { return "20"; }
};

obj + 5; // 15
Reminder: For ordinary object-to-primitive conversion, JavaScript may try valueOf() before toString(), depending on the preferred type.

5. Special Cases

NaN === NaN;          // false
Object.is(NaN, NaN); // true

null == undefined;   // true
null === undefined;  // false

[] + [];  // ""
[] + {};  // "[object Object]"
{} + [];  // 0

6. BigInt

1n + 1;   // TypeError
1n == 1;  // true
1n === 1; // false

Important: BigInt and Number do not mix in arithmetic.

7. Symbol

const s = Symbol("x");

String(s); // "Symbol(x)"
+s;        // TypeError

Symbols can be stringified explicitly, but numeric coercion fails.

8. Best Practices

  • Prefer ===
  • Use explicit conversions (Number, String, Boolean)
  • Avoid implicit coercion with +
  • Use Number.isNaN
  • Use value == null for null/undefined checks
Practical rule: Be explicit unless you intentionally want coercion.
